    Pyc

    • русский

    When we import the python script, we will generate a corresponding pyc file in the directory, which is a persistent storage form of pythoncodeobj to speed up the next load.

    File Structure

    Pyc file consists of three major parts

    • The first 4 bytes is a Maigg int, which identifies the version information of this pyc.
    • The next four bytes are still an int, which is the time that pyc is generated.
    • Serialized PyCodeObject, structure reference include/code.h, serialization method python/marshal

    pyc full file parsing can refer to

    • How the Python program is executed
    • PYC File Format Analysis

    About co_code

    A string of binary streams, representing the sequence of instructions, specifically defined in include/opcode.h , or refer to python Opcodes.

    by

    • The directive (opcode) is divided into parameters and no parameters, which are divided into https://github.com/python/cpython/blob/fc7df0e664198cb05cafd972f190a18ca422989c/Include/opcode.h#L69
    • parameter (oparg)

    The above parameters of python3.6 always occupy 1 byte. If the instruction has no parameters, it is replaced by 0x00, which is ignored by the interpreter during the running process. It is also the technical principle of Stegosaurus; the version is lower than python3.5. The middle instruction has no arguments but no 0x00 padding.

    Tools

    pycdc

    Info

    Convert python bytecode to readable python source code, including disassembly (pycads) and decompilation (pycdc)

    Stegosaurus

    Info

    Allows us to embed any Payload in a Python bytecode file (pyc or pyo). Due to the low coding density, the process of embedding Payload does not change the running behavior of the source code, nor does it change the file size of the source file.

    The principle is to use the redundant space in the python bytecode file to hide the complete payload code into these fragmented spaces.
